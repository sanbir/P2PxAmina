How it works

Tokenised tri-party repo. Cash on one side, eligible institutional collateral on the other, AMINA Bank as agent in the middle.

01
Onboard
AMINA performs KYB and credit screening. Custody account linked via approved providers (Fireblocks, AMINA, BitGo, Anchorage, Fordefi).
02
Place order
Submit lend or borrow order with principal, term and collateral type. Rate fixed at AMINA's quarterly review.
03
Match and sign
Off-chain engine matches counterparties pseudonymously. Both sides sign EIP-712 batch terms with AMINA co-signature.
04
Settle and monitor
Settlement at custodian, T+0. Position accrues interest. AMINA monitors coverage and executes liquidations if required.

Built for institutional balance sheets

P2PxAmina applies banking-grade legal, regulatory and operational structure to tokenised repo.

Tri-party agent

AMINA Bank AG holds collateral in segregated escrow, manages valuation, issues margin calls and executes liquidations under FINMA supervision.

Pseudonymous counterparties

All counterparties are KYB-verified by AMINA but their identities are not disclosed to each other. Exposure is to the collateral pool, not to a named party.

SFTR reporting included

All transactions reported under Securities Financing Transactions Regulation. Daily MTM, full audit trail, downloadable confirmations.

Glossary
Repo Rate
Fixed interest rate on the repurchase agreement, quoted p.a. ACT/360 day-count convention.
Coverage Ratio
Collateral market value divided by outstanding balance. Primary risk metric used by AMINA.
LTV
Loan-to-value: principal advanced as percentage of collateral value at inception.
Margin Call
Formal request to post additional collateral when coverage falls below threshold. 48-hour cure period.
Health Factor
Secondary DeFi-native metric: coverage ratio expressed as multiple of liquidation threshold.
EIP-712
Standard for human-readable typed data signing on Ethereum, used for off-chain deal authorisation.

User stories
Lender
  1. See current rate and depth before committing.
  2. Enter amount/term and see net yield immediately.
  3. Link custody once; subsequent orders flow automatically.
  4. See aggregate position (one number) across N bilateral deals.
  5. Receive match notification with review window.
  6. Review EIP-712 in human-readable form before signing.
  7. See yield accruing in real time.
  8. Receive maturity notification 7 days ahead.
  9. See collateral type and LTV backing my exposure.
  10. Download statement for fund admin.
Borrower
  1. Enter collateral and see max borrowable instantly.
  2. See borrow rate and liquidity before submitting.
  3. Confirm LTV and HF at deal inception.
  4. See outstanding balance + accrued interest live.
  5. Receive 48h margin warning before liquidation.
  6. One-click top-up flow without calling a desk.
  7. Make partial repayments any time.
  8. See exact repayment amount before confirming.
  9. See liquidation sequence clearly (warn → partial → full).
  10. Receive any surplus collateral automatically after full liq.
AMINA Risk Desk
  1. Real-time portfolio health dashboard.
  2. Per-deal HF colour-coded for distress detection.
  3. Oracle feed status with CAPO/breaker state.
  4. Cap utilisation across all 9 dimensions.
  5. Audit log of every privileged AMINA OPS action.
Known limitations
  • No real persistence — all state resets on reload.
  • No actual EIP-712 cryptography; signing modal is illustrative only.
  • Mobile (<768px) is best-effort; primary target is desktop >= 1280px.
  • Charts are static SVG illustrations, not data-driven.
  • AMINA OPS execute buttons are deliberately disabled in this Risk Desk view per role separation.
  • KYB document uploads are visual placeholders only — no file handling.
  • Print/PDF statement is a visual preview, not a generated PDF.
